# محرّك التدقيق الذكيّ — الطبقة 3 (لام)

> يرفع جودة ربط الأنظمة من «جيّد» إلى «دقيق 100%» عبر قراءة GPT لكلّ حدث.

## ماذا يفعل؟

يمرّ على الروابط بين الأحداث والأنظمة (المبنيّة في قاعدة البيانات)، ولكلّ رابط:
1. يؤكّد: هل هذا **تعديل حقيقيّ** للنظام أم مجرّد ذكر عابر؟ (يصفّي الضوضاء)
2. يصحّح **التاريخ الهجريّ** المشوّه.
3. ينقّي **المواد المعدّلة** (يحذف الأرقام الخاطئة).
4. يكتب **ملخّصًا دقيقًا**: ماذا عدّل هذا القرار؟

النصّ الحرفيّ (`raw_text`) **لا يُمسّ** — القاعدة الدستوريّة مصونة.

## خطوات التشغيل (مثل سكربت أم القرى)

### 1. تجهيز المفاتيح
- انسخ `config.example.py` باسم `config.py`.
- املأ:
  - `OPENAI_API_KEY`: من platform.openai.com (تحتاج رصيدًا — التكلفة ~2-3$ للكلّ).
  - `SUPABASE_SERVICE_KEY`: نفس مفتاح service_role المستخدم سابقًا.

### 2. التثبيت (مرّة واحدة)
افتح PowerShell في مجلّد `lam-layer3` واكتب:
```
pip install openai requests
```

### 3. التجربة على 10 أوّلًا
```
python refine.py 10
```
راقب: هل يطبع «✓ تعديل حقيقيّ» و«— ذكر عابر» بشكل منطقيّ؟

### 4. التشغيل الكامل
```
python refine.py
```
يمرّ على كلّ الروابط (~540). يأخذ دقائق. يُستأنف إن انقطع (يتخطّى المدقّق).

## بعد الاكتمال

البيانات المصحّحة تظهر تلقائيًّا في واجهة العرض (الأنظمة وتسلسلها) — التواريخ صحيحة، الذكر العابر مُصفّى، المواد دقيقة.

## التكلفة
~540 حدثًا × استدعاء GPT صغير (gpt-4o-mini) = **~2-3 دولار** لمرّة واحدة.
